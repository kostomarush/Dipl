Описание режима TCP

Режим сканирования TCP в Nmap представляет собой полное сканирование TCP, которое использует три TCP-пакета (синхронизации, подтверждения и сброса) для установления состояния портов на целевом хосте. Этот режим сканирования представляет собой стандартный TCP-скан, который отправляет полные установочные пакеты для определения статуса портов.

Ниже представлены основные характеристики и особенности режима сканирования -sT:

1. Основные характеристики:

Отправляет полные установочные пакеты (SYN, ACK, RST) для каждого сканируемого порта.

Использует стандартные значения портов, которые требуется проверить.

2. Принцип работы:

Отправляет TCP-пакет с установкой флага SYN. Если порт открыт, хост ответит SYN/ACK. Если порт закрыт, хост ответит RST. После получения ответа на пакет с установкой SYN, Nmap отправляет пакет с установкой флага RST для закрытия соединения. Выводит результаты, основанные на ответах от целевого хоста.

3. Преимущества:

Прост в использовании и может работать с большинством сетевых устройств и хостов. Эффективен для определения статуса открытых и закрытых портов.

4. Недостатки:

Медленнее по сравнению с более "легкими" методами сканирования, такими как SYN сканирование. Может быть легко обнаружено системами обнаружения вторжений (IDS), так как включает в себя полный обмен TCP-пакетами. Режим сканирования -sT полезен, когда требуется точное и надежное определение статуса портов, а скорость сканирования не является критическим фактором. Важно учитывать, что использование данного режима может быть заметно на сетевой активности и может привлечь внимание средств обнаружения вторжений.
